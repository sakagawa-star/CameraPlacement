# 要求仕様書: F12 配置パターン定義

## 1. プロジェクト概要

- **何を作るか**: 病室内の6台カメラの代表的な配置パターン（コーナー型、壁面均等型、天井格子型、ベッド集中型、ハイブリッド型）をプリセットとして定義し、Camera オブジェクトのリストを生成するモジュール
- **なぜ作るか**: F13（配置比較・レポート）で複数の配置パターンを比較評価するために、手動設計の代表的な配置を定義する必要がある。各パターンの特性（カバレッジ、三角測量角度など）を比較し、最適化（F14-F15）の初期解やベースラインとしても使用する
- **誰が使うか**: F13（配置比較モジュール）、F15（最適化エンジンの初期解）、開発者
- **どこで使うか**: Python 3.12 ローカル環境

## 2. 用語定義

| 用語 | 定義 |
|------|------|
| プリセット (preset) | 6台のカメラ位置・注視点を事前定義した配置パターン。`PlacementPreset` インスタンスとして表現する |
| カメラ設定 (camera config) | 1台のカメラの位置と注視点のペア。`CameraConfig` インスタンスとして表現する |
| プリセット名 (preset name) | プリセットを識別する英語の文字列。`"upper_corners"` のようにスネークケースで記述する |
| カメラ設置可能領域 (camera zone) | カメラを設置できる空間。X: 0.2〜2.6 m、Y: 0.2〜3.3 m、Z: 0.2〜2.3 m |
| 注視点 (look_at) | カメラが向く先のワールド座標。`Camera.look_at` パラメータに対応する |
| バリデーション (validation) | カメラ位置がカメラ設置可能領域内にあるか、Camera オブジェクトが正常に生成できるかの検証 |
| `create_camera()` | F02 で定義されたカメラ生成関数。position, look_at を受け取り `Camera` インスタンスを返す |

## 3. 機能要求一覧

### FR-01: プリセットデータの定義

5種類の配置パターンをプリセットとして定義する。各プリセットは6台のカメラの位置と注視点を含む。

- 入力: なし（ハードコードされた定義）
- 出力: 5つの `PlacementPreset` インスタンス
- プリセット一覧:

| プリセット名 | 日本語名 | 配置戦略 |
|---|---|---|
| `upper_corners` | 上部コーナー型 | カメラ設置可能領域の上部4隅 + 短辺中央2箇所に配置。全カメラ Z=2.3m |
| `wall_uniform` | 壁面均等型 | 左右壁面に各2台 + 前後壁面に各1台。壁面周囲に均等に分散配置。全カメラ Z=2.3m |
| `overhead_grid` | 天井格子型 | 天井直下に2×3の格子状に配置。全カメラ Z=2.3m。真上からの俯瞰視点 |
| `bed_focused` | ベッド集中型 | ベッド周囲を重点的にカバーする配置。注視点はベッド中央 |
| `hybrid` | ハイブリッド型 | 上部4隅で広範囲カバレッジ + 壁面中段2台でベッドの角度改善 |

- 受け入れ基準: 5つのプリセットが定義され、各プリセットの `camera_configs` が6要素のタプルであること。全カメラ位置がカメラ設置可能領域内（X: 0.2〜2.6, Y: 0.2〜3.3, Z: 0.2〜2.3）であること

### FR-02: プリセット取得（名前指定）

プリセット名を指定して1つのプリセットを取得する。

- 入力: `name: str`（プリセット名）
- 出力: `PlacementPreset` インスタンス
- 存在しない名前の場合: `KeyError` を送出する。エラーメッセージに利用可能な名前一覧を含める
- 受け入れ基準: `get_preset("upper_corners")` で `PlacementPreset` が返ること。`get_preset("invalid")` で `KeyError` が送出されること

### FR-03: 全プリセット取得

定義済みの全プリセットをリストで取得する。

- 入力: なし
- 出力: `list[PlacementPreset]`（5要素）
- 順序: `upper_corners`, `wall_uniform`, `overhead_grid`, `bed_focused`, `hybrid` の定義順
- 受け入れ基準: 長さ5のリストが返り、各要素が `PlacementPreset` であること

### FR-04: プリセット名一覧取得

定義済みの全プリセット名をリストで取得する。

- 入力: なし
- 出力: `list[str]`（5要素）
- 順序: FR-03 と同一
- 受け入れ基準: `["upper_corners", "wall_uniform", "overhead_grid", "bed_focused", "hybrid"]` が返ること

### FR-05: Camera オブジェクト生成

プリセットから `Camera` オブジェクトのリストを生成する。

- 入力:
  - `preset: PlacementPreset`（必須）
  - `room: Room | None`（任意、デフォルト None）
- 出力: `list[Camera]`（6要素）
- room が None の場合: バリデーションなしで Camera を生成する
- room が非 None の場合: 全カメラ位置が `room.camera_zone` 内にあることを検証する。1台でも範囲外の場合は `ValueError` を送出する。エラーメッセージに範囲外のカメラインデックス（0始まり）と座標を含める
- Camera の生成には `create_camera()` 関数を使用する。`up_hint` と `intrinsics` はデフォルト値を使用する
- 受け入れ基準: 各プリセットから6台の `Camera` オブジェクトが生成でき、各カメラの `position` と `look_at` がプリセット定義と一致すること。`room` 指定時に範囲外の位置があれば `ValueError` が送出されること

## 4. 非機能要求

### パフォーマンス

- `get_preset` の実行時間: 1ms 以内（辞書ルックアップのみ）
- `create_cameras` の実行時間: 6台のカメラ生成で 1ms 以内

### 対応環境

- Python 3.12
- numpy >= 2.4.2（既存依存）

### 信頼性

- 該当なし。永続データを扱わず、全データがコード内にハードコードされているため

## 5. 制約条件

- 使用ライブラリ: numpy（既存依存のみ。追加のライブラリは不要）
- F01（`Room`, `AABB`）、F02（`Camera`, `create_camera`）の既存インターフェースを変更しない
- 全プリセットのカメラ台数は6台固定（CLAUDE.md の仕様に準拠）
- プリセットの座標はデフォルト病室（2.8 × 3.5 × 2.5 m）に合わせて設計する
- 全カメラ位置はカメラ設置可能領域内（X: 0.2〜2.6, Y: 0.2〜3.3, Z: 0.2〜2.3）であること
- 全カメラの `forward` ベクトルがデフォルト `up_hint`（[0, 0, 1]）と平行にならないこと（Camera の `__post_init__` バリデーションを通過すること）

## 6. 優先順位

| 要件 | MoSCoW |
|------|--------|
| FR-01 プリセットデータの定義 | Must |
| FR-02 プリセット取得（名前指定） | Must |
| FR-03 全プリセット取得 | Must |
| FR-04 プリセット名一覧取得 | Must |
| FR-05 Camera オブジェクト生成 | Must |

## 7. エッジケースの期待動作

| ケース | 期待動作 |
|--------|---------|
| `get_preset("invalid")` | `KeyError` を送出。メッセージに利用可能な名前一覧を含む |
| `get_preset("")` | `KeyError` を送出 |
| `create_cameras` に room=None を渡す | バリデーションなしで Camera 生成。正常動作 |
| `create_cameras` にデフォルト Room を渡す | 全プリセットでバリデーション通過（全位置が設置可能領域内のため） |
| `create_cameras` にカメラ設置可能領域が狭い Room を渡す | 範囲外の位置があれば `ValueError` |

## 8. スコープ外

- ユーザーによるカスタムプリセットの登録・保存機能（Camera リストを直接作成すればよい）
- 設定ファイル（TOML/YAML/JSON）からのプリセット読み込み
- パラメトリックな配置生成（例: 角度パラメータから自動配置）
- 7台以上のカメラへの対応
- デフォルト病室以外の部屋サイズへのプリセット自動スケーリング
