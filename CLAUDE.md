# CLAUDE.md - カメラ配置最適化プロジェクト

## プロジェクト概要

病室内でのマーカレスモーションキャプチャ（Pose2Sim）による3Dキーポイント推定の精度を最大化するために、産業用カメラの最適な配置を計算するツールを開発する。

### 背景

- 2D→3Dリフトアップの学習データとして、2Dキーポイントと3Dキーポイントのペアデータが必要
- 2D・3DキーポイントはPose2Simで推定する
- 学習データの品質を上げるため、3Dキーポイント推定の精度を最大化したい
- そのためにはカメラ配置が重要であり、本プロジェクトはその配置を計算・評価するツールを作る

### 最終目標

**各キーポイントが常に3台以上のカメラから観測される**カメラ配置を求める。
加えて、三角測量の精度を最大化する配置を求める。

## 開発ルール

### 開発方針

- **シンプルな機能を一つずつ作り、積み重ねて目的を達成する**
- 大きな機能を一度に作らない。小さく作って動作確認し、次の機能へ進む

### 機能ごとの開発フロー

各機能について、以下のフローを**厳守**する。**planモードは使わない**（通常モードで調査・計画を行う）。

1. **調査・計画** → 通常モードで既存コードを調査し、要求仕様書と機能設計書を設計する
2. **ドキュメント保存** → 要求仕様書と機能設計書を `docs/` にファイル保存する。**保存が完了するまで実装に進んではならない**
3. **レビュー（Subagent + 人）** → `docs/` に保存されたドキュメントをSubagent（Agentツール）でレビューする。ユーザーも同時にレビューする。レビュー実行時は `docs/REVIEW_CRITERIA.md` の基準に従うこと
4. **修正（必要な場合）** → レビューで問題があれば、再調査してドキュメントを更新する。**ステップ1〜3を問題がなくなるまで繰り返す**
5. **引き継ぎ・/clear** → CLAUDE.mdの「現在の進捗」セクションを更新し、実装セッションに必要な情報を整える。その後ユーザーが `/clear` を実行
6. **実装** → ドキュメント（要求仕様書・機能設計書・CLAUDE.md）を読んで実装

### ドキュメント作成ルール

- **実装前に必ず「要求仕様書」と「機能設計書」を作成し、ファイルに保存すること**
- ドキュメントが保存されていない場合は、**実装を中止**する
- 要求仕様書：何を達成すべきか（入出力、制約、品質基準）。作成時は `docs/REQUIREMENTS_STANDARD.md` の基準に従うこと
- 機能設計書：どう実現するか（モジュール構成、アルゴリズム、データ構造）。作成時は `docs/DESIGN_STANDARD.md` の基準に従うこと
- ドキュメントは `docs/` ディレクトリに置く
- **/clear 後でも実装がスムーズにできるよう、必要な情報を全て記述する**
- 暗黙知に頼らず、**自己完結したドキュメント**にする（前の会話コンテキストがなくても実装できること）

  - ファイル名：`{管理番号}_test_result.txt`（例：`F01_test_result.txt`）
  - 内容：pytestの `-v` 出力をそのまま保存する

## ハードウェア仕様

### カメラ：DFK33GR0234（The Imaging Source）

- センサー：1/2.6型 onsemi AR0234CS CMOS
- 解像度：1920 × 1200（2.3 MP）
- ピクセルサイズ：3 µm × 3 µm
- センサー物理寸法：5.76 mm（H）× 3.6 mm（V）
- フレームレート：最大 50 fps
- シャッター：グローバルシャッター
- インターフェース：GigE（PoE）

### レンズ：LM3NC1M（興和オプトロニクス）

- 焦点距離：3.5 mm
- マウント：Cマウント（C-CS変換リングでカメラに装着）
- 対応センサーサイズ：1/1.8型まで
- 絞り：F2.4 ～ F14
- TV歪曲：0.4% 以下

### 実効画角（1/2.6型センサー × 3.5mmレンズ）

計算式：`FOV = 2 × arctan(sensor_dim / (2 × focal_length))`

- 水平FOV：≈ 79°
- 垂直FOV：≈ 54°
- 対角FOV：≈ 88°

### カメラ台数

- **6台固定**（8台対応は時間的コストが大きいため対象外）

## 撮影環境

### 座標系

```
        Y（奥行 3.5m）
        ^
  3.5   |================= 奥壁 =================|
        |                                         |
  3.3   |  - - - 設置可能領域上限 (Y) - - - - -   |
        |                                         |
        |          ┌─────────┐                    |
        |          │  ベッド  │ 1m×2m              |
        |          │ X:0.9-1.9│                    |
        |          │ Y:1.5-3.5│                    |
  1.5   |          └─────────┘                    |
        |                                         |
        |      （歩行エリア）                      |
        |                                         |
  0.2   |  - - - 設置可能領域下限 (Y) - - - - -   |
        |                                         |
  0     +---------> X（幅 2.8m）
       0.0  0.2                            2.6  2.8

       Z = 高さ（上向き、0 ～ 2.5m）
```

- 原点：部屋の床面、手前左コーナー
- X軸：幅方向（0 ～ 2.8 m）
- Y軸：奥行方向（0 ～ 3.5 m）
- Z軸：高さ方向（0 ～ 2.5 m）

### 部屋

- 寸法：2.8 m × 3.5 m × 2.5 m（幅 × 奥行 × 高さ）

### ベッド

- 寸法：1.0 m × 2.0 m × 0.2 m（幅 × 奥行 × 高さ）
- X方向：0.9 m ～ 1.9 m（部屋中央）
- Y方向：1.5 m ～ 3.5 m（頭側が奥壁に接触）
- Z方向：0 ～ 0.2 m

### カメラ設置可能領域

壁・天井には直接設置できない。壁際にポールを立てて設置する。壁・天井から20cm以上の間隔が必要。

- X：0.2 ～ 2.6 m（幅 2.4 m）
- Y：0.2 ～ 3.3 m（奥行 3.1 m）
- Z：0.2 ～ 2.3 m（高さ 2.1 m）

実運用ではポール上部に設置するため、Z = 1.8 ～ 2.3 m 付近が現実的。

### 撮影対象

- 身長：177 cm、体重：80 kg の男性
- 動作パターン：
  - 部屋内の歩行（ベッド以外の床面エリア）
  - ベッドに座る（座位）
  - ベッドに寝る（仰臥位、側臥位）

### 活動ボリューム（キーポイントが存在しうる空間）

| 動作 | Z範囲 | XY範囲 |
|---|---|---|
| 歩行 | 0 ～ 1.8 m | ベッド以外の床面 |
| ベッド上座位 | 0.2 ～ 1.1 m | ベッド上 (X:0.9-1.9, Y:1.5-3.5) |
| ベッド上臥位 | 0.2 ～ 0.5 m | ベッド上 |

## 制約条件まとめ

1. **カバレッジ制約**：活動ボリューム内の全キーポイントが3台以上のカメラから同時に視認可能
2. **設置制約**：カメラは設置可能領域内にのみ配置可能
3. **台数制約**：6台固定
4. **物理制約**：ベッドによるオクルージョン、被験者自身のセルフオクルージョン

## 品質指標（三角測量精度に影響する要素）

1. **カメラ間の角度分離**：同一点を観測する複数カメラの視線角度差。90°前後が理想。15°未満だと精度大幅低下
2. **2D検出精度**：カメラから遠すぎると被写体が小さくなり2Dキーポイント検出精度が低下
3. **均一なカバレッジ**：空間全体でカバレッジが偏らないこと
4. **歪み**：画像端部では歪みにより2D精度が低下する可能性あり（ただしTV歪曲0.4%と小さい）
